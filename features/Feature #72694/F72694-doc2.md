## 1 Feature Summary

### What / Why

- **Goal**：解决多绑定（multi-binding）轴场景下轴标签被“分离/散开”导致的可读性问题，让用户无需写 Script 也能一键把标签放到对侧。
- **User entry**：选中轴标签区域 → 右键 → `Axis Properties` → `Label` → 勾选 `Labels on Opposite Side`。
- **Key behavior**：
  - X 轴：bottom ↔ top
  - Y 轴：left ↔ right
  - 多绑定轴：勾选后标签应“合并到对侧”，不再出现分离渲染；取消勾选恢复原侧
  - 多轴/双轴：每条轴独立生效，互不串扰；避免左右轴标签重叠或错位

### Scope & Non-scope

- **In scope**：
  - Axis-based（标准 X/Y 坐标轴）：Bar/Column/Line/Area/Bubble/Step Line/Waterfall
  - Multi-axis（双 Y 轴/组合图）：Bar（双轴）、Line（双轴）、Area（双轴）、Combo
  - Financial：Candle/Stock（行为同标准 X/Y，主要验证不回归）
  - 其他特殊：
    - Funnel: Y轴只能绑定D,选中之后到对侧.  X轴绑定M时, 无轴. 绑定D,应用到对侧.
    - Gantt: 单独时间轴, 与X|Y轴不同, 默认在X轴上, 支持属性. 选中之后应用到对侧
- **Out of scope（不应支持/不应暴露入口）**：
  - Polar：Radar（极坐标不支持）
  - Partition：Pie/Donut/Treemap/Sunburst（无轴）
  - Network：Network/Hierarchy（无轴）
  - Geographic：Map/Contour Map（无轴）
  - None：Tree/Network/Circular Network/Scatter Contour 等（无轴）
  - 其他特殊：
    - Pareto：本身双 Y 轴, 右边的Y轴代表百分比(不支持) Bug #74009

## 2 Functional Rules（规则化描述，避免只写步骤）

> 用“默认 + 变化 + 约束 + 优先级”的方式描述规则，便于写用例和定位回归。

- **Default behavior**：checkbox 默认未勾选；轴标签显示在默认侧（X=bottom，Y=left；双轴按各自默认侧）。
- **Toggle / config behavior**：
  - 勾选：该轴标签移动到对侧（X：top；Y：right）
  - 取消：恢复默认侧
  - 保存/刷新/重新打开对话框：checkbox 状态正确回显；渲染位置一致
- **Multi-object interaction（多轴/多绑定）**：
  - 术语：
    - **多 M 轴**：同一方向的轴（X 或 Y）上 **绑定了多个 Measure**（多个 M）。
    - **多 D 轴**：同一方向的轴（X 或 Y）上 **绑定了多个 Dimension**（多个 D）。
    - **secondary axis**：该方向存在“对侧轴”（例如 Y=Right 或 X=Top）并启用/显示。
  - 多 M 轴（多 Measure 绑定）的出现/不出现规则（需要明确验证 Swap X/Y）：
    - **未启用任何 secondary axis（该方向所有轴均在同一侧）**：支持显示该 checkbox；勾选后该方向**所有相关轴标签**整体移动到对侧。
    - **任意轴启用了 secondary axis（该方向两侧同时存在轴）**: 该方向**不支持**（`Labels on Opposite Side` 不出现，或出现但不可用）；避免出现 Top&Bottom / Left&Right 两侧同时控制导致的冲突。Bug #74047 / Bug #74098。
  - 多 D 轴（多 Dimension 绑定）：
    - **默认行为**：标签默认显示在对侧。
    - **勾选后行为**：标签切换对侧。
- **Script Usage（脚本用法）**：
  - 新增属性：`labelOnSecondaryAxis`
  - 常见用法（按场景拆分，便于用例覆盖）：

  ```javascript
  // 1) 全局轴（本次 Bug 2 修复的核心路径）
  Chart1.xAxis.labelOnSecondaryAxis = true; // X 轴标签 → top
  Chart1.yAxis.labelOnSecondaryAxis = true; // Y 轴标签 → right

  // 2) 单图（single chart）且该方向已启用 secondary axis（两侧同时有轴）
  //    结论：从 UI 规则推断不应该支持；此处应重点验证“不抛错 + 不破坏轴结构”。
  Chart1.y2Axis.labelOnSecondaryAxis = true; // 不建议：疑似不支持（是否应提 bug / 或忽略该脚本？） //暂时忽略, 不影响
  Chart1.yAxis.labelOnSecondaryAxis = true;  // 可能触发与 UI 同类问题：axis 丢失/异常（需验证）

  // 3) 指定具体 axis 对象
  axis.STATE.labelOnSecondaryAxis = true; // 期望：UI 可同步回显（Bug #74098）
  axis["Sum(CUSTOMER_ID)"].labelOnSecondaryAxis = true; // 现象：生效但 UI 不同步/配置丢失（Bug #74098）

  // 4) 读取当前值
  var flag = Chart1.yAxis.labelOnSecondaryAxis; // true / false
  ```

  - 备注（测试角度）：
    - **脚本容错优先**：允许“UI 不回显/不支持”，但至少应做到 **不抛错**、不导致渲染崩溃。
    - **覆盖策略**：不同 chart type + 多轴场景走通 1 条主路径，确保脚本设置/读取不报错即可。

- **Priority rules（UI vs Script）**：
  - 当通过 Script 设置轴到对侧时（例如 `AxisSpec.setAxisStyle(AxisSpec.AXIS_SINGLE2 / AXIS_DOUBLE2 ...)` 或 `*.labelOnSecondaryAxis = true`）：
    - **优先级**：以 Script 为准，UI checkbox 不应强行覆盖脚本结果，避免状态叠加/来回抖动。
    - **容错要求**：脚本执行 **不抛错、不导致渲染异常或轴结构丢失**，即使 UI 暂时不能完整回显。
    - **状态同步**：在支持的场景中，`labelOnSecondaryAxis` 设置后，UI 中的 `Labels on Opposite Side` 状态应与脚本保持一致；若因已知 Bug 暂时无法同步，需要在测试结论中明确标注。
- **Error handling / guardrails**：
  - 不支持类型：不应暴露 Axis Properties / Label 页或不包含该 checkbox
  - 若误暴露：勾选不应崩溃、不乱码、不导致轴布局错乱

## 4 Test Focus（只列必须测的路径）

### P0 - Core Path（必测主路径）

- **基础单轴行为**：  
  - Bar/Column（axis-based 核心）：X/Y 单绑定下勾选/取消，标签在默认侧与对侧之间正确切换，无异常渲染。
  - UI 状态回显：同一图表分别打开 X/Y 的 `Axis Properties → Label`，checkbox 状态独立且一致。
- **持久化与回放**：  
  - 勾选后保存 Dashboard → 刷新/重新打开 → 导出 vsml → 再导入，checkbox 状态与渲染结果完全一致，不丢失。
- **Script 基础路径**：  
  - 使用 `Chart1.xAxis/yAxis.labelOnSecondaryAxis = true/false` 设置，再通过 UI 打开对话框检查：脚本不抛错，渲染稳定，UI 在支持场景下能正确同步状态。

### P1 - Functional / Boundary（高风险功能与边界）

- **多绑定 / 多轴 / secondary axis 相关**：  
  - 多 M / 多 D 绑定：axis-based 图表下多绑定场景勾选后标签“合并到对侧”，不出现分离；取消后恢复默认布局。  
  - 双轴 / Combo：Left/Right 分别勾选与同时勾选时标签不重叠、不混叠，轴结构不丢失。  
  - secondary axis 限制：同一方向启用 secondary axis 后，对应方向应禁用/不支持该选项；若 UI 仍可操作，验证属性实际无效且不破坏布局（关联 Bug #74047 / #74098）。
- **特殊图表行为（Funnel / Gantt / Pareto）**：  
  - Funnel：验证 Y 轴仅 D 绑定、X 轴 D 绑定的行为是否符合“对侧显示”的语义；X 轴 M 绑定无轴时不支持。  
  - Gantt：时间轴支持该属性；勾选后移到对侧，刷新/保存后状态与渲染保持一致。  
  - Pareto：确认该选项不应支持（或为禁用态）；若可勾选，则重点验证 axis 不丢失并记录为 Bug #74009 相关。
- **不支持类型负向验证**：  
  - Radar/Map/Pie/Network/Tree/Scatter Contour 等不应暴露 Axis Properties / Label 或该 checkbox；若 UI 误暴露，只做“不可用但不崩溃/不乱码/不乱布局”的稳定性验证。
- **Locale / 文案**：  
  - 在中文/日文等 locale 下检查 `Labels on Opposite Side` 文案显示正常，有翻译或英文 fallback，不出现 key 原文或乱码。

### P2 - Extended（按需抽样）

- **Export / Print / Embedded**：  
  - 对典型 axis-based（Bar/Column）和 multi-axis 场景抽样做 PDF/图片导出、嵌入 Report，确认对侧标签在导出/嵌入后位置与样式保持一致。
- **Performance**：  
  - 多绑定/双轴下频繁勾选/取消、打开/关闭对话框无明显卡顿；无需专项压测，仅做主路径体感检查。
- **Backward compatibility & Script 兼容**：  
  - 旧 vsml（无 `labelOnSecondaryAxis`）导入后默认 false，行为与旧版本一致。  
  - 对已有使用 `setAxisStyle(AxisSpec.AXIS_SINGLE2/DOUBLE2)` 的脚本，验证 Script 优先且 UI 不造成异常（UI 同步可不强求，但不能破坏渲染或轴结构）。

## 6 Special Testing（仅当涉及时填写）

### Security

- 无新增外部输入面；常规回归即可。

### Performance

- 无专项；抽样多绑定/双轴下打开对话框与切换不明显卡顿。

### Compatibility

- **Old config**：旧 vsml 不含 `labelOnSecondaryAxis` 时默认 false，行为与旧版一致。
- **Round-trip**：勾选后保存/导出/导入不丢状态（至少覆盖 Bar/Column 各 1）。

### Script / Programmatic Support

- **API(s)**：`chart.getAxis("Bottom/Left/Right").setAxisStyle(...)`、`AxisSpec.AXIS_SINGLE2 / AXIS_DOUBLE2`。
- **UI vs Script priority**：测试-分析结论：**Script 优先，UI 设置无效**（需验证“无效但稳定”）。
- **Gotchas**：关注多次切换/多轴下是否出现轴丢失、轴乱、属性无效但可操作等（见 Bug/Issue 列表）。

### Localization

- **Strings / keys**：`Labels on Opposite Side`。
- **Fallback behavior**：无翻译时应英文 fallback，不显示 key 原文/乱码（测试-分析：初步 ok）。

### Docs / API

- 若对外脚本文档涉及 `getAxisStyle()/setAxisStyle()`，补充 UI 与 Script 优先级说明（至少在测试备注中固化）。
