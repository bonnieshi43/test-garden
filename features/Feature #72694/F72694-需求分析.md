### 1. 特定功能理解与范围界定

#### 核心目标总结
需求要求在图表轴标签配置中添加一个直观的用户界面功能，允许用户通过右键菜单直接调整轴标签的位置（上/下/左/右），而无需编写脚本修改 `AxisSpec.AxisStyle`。

- **痛点**：当轴有多个绑定时，轴标签可能分离或重叠             📅 **测试-重要**：这个部分需要重点关注

- **新功能目标**：提供 UI 级属性面板，使非技术用户可直接配置标签位置

#### 涉及模块识别
- **前端**：图表属性面板（Portal 中的图表设计界面，可能涉及 Angular Material 对话框）
- **后端**：轴对象序列化/反序列化、`AxisSpec` 配置管理
- **数据层**：配置持久化（报表或仪表板中的轴配置）
- **脚本**：需考虑 Script API 是否需要支持该新属性的读写

#### 类型判断
**UI + 配置数据混合型需求**  
- UI 变更（新增属性控件）
- 数据模型扩展（轴标签位置属性）
- 向后兼容性要求（已有配置必须保持可读）
