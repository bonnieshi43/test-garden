# 1. Feature Overview

> 仅在本功能存在对应内容时保留本节；如不适用，可整体省略本节。

## Feature Description

`Labels on Opposite Position`（UI 文案可能为 “Labels on opposite side”）用于在**无需脚本**的前提下，通过 Axis Properties 面板，把轴标签（axis labels）从当前一侧移动到对侧：

- X 轴：在 `Top` / `Bottom` 之间切换
- Y 轴：在 `Left` / `Right` 之间切换

该属性本质上是对 `AxisSpec.AxisStyle` / `labelPosition` 的 UI 封装，目标是：

- 解决多轴、多绑定场景下标签遮挡、分离、不易阅读的问题
- 降低配置成本，避免用户编写脚本 `AxisSpec.setAxisStyle(...)`

## Scope

* **适用模块**：StyleBI Dashboard 设计器中的 Chart 设计
* **适用组件**：所有带 X/Y 轴的 Cartesian / Financial 图表（Bar/Line/Area/Combo/Scatter/Bubble/Waterfall/Pareto/部分 Candle 等）
* **适用对象**：
  * X 轴 / Y 轴（主值轴 / 次值轴）
  * 单轴、多轴（多 Y 轴）场景
* **不适用对象**（应被排除）：无轴或特殊轴图表，如 Pie/Donut、Map、Treemap、Network、Gauge、Gantt、部分 Radar
* **适用运行环境**：仅面向 **桌面浏览器设计器**；Mobile 端当前**不支持编辑 Axis Properties**，本特性在 Mobile 只需验证显示/渲染结果，无需编辑入口

# 2. Functional Rules

> 仅在本功能存在明确规则时保留本节；如无专门规则，可省略本节。

## 2.1 默认行为（Default Behavior）

- 旧报表中未显式配置 `Labels on Opposite Position` 时：
  - 轴标签位置保持与旧版本一致（X：底部，Y：左侧），功能默认 **不改变** 既有视觉行为
- 新建报表时：
  - 默认值与当前版本未开启该属性时的行为一致（不强制反转）
  - `Labels on Opposite Position` 属性的**默认值为 `false`**（即标签保持在“非对侧”的默认位置）

## 2.2 属性与 UI 规则（UI & Property Rules）

- 属性位置：
  - 暴露在 Axis Properties 面板中，作为布尔型/选项型属性 `Labels on Opposite Position`
- 行为语义（X/Y 轴一致）：
  - 当属性 **未选中**（`false`）时，轴标签位于该轴的**默认侧**：
    - X 轴：默认在 `Bottom`
    - Y 轴：默认在 `Left`
  - 当属性 **选中**（`true`）时，轴标签移动到**相反一侧**，并保持该位置，直到用户再次修改：
    - X 轴：`Bottom → Top` 或 `Top → Bottom`
    - Y 轴：`Left → Right` 或 `Right → Left`
  - 属性本身不直接暴露 “Top/Bottom/Left/Right” 的枚举列表，而是以“是否移动到对侧”的方式工作
- 特殊轴 / 不适用图表类型：
  - 对于无轴或特殊轴（Radar、Pie/Donut、Map、Gauge、Network、Gantt 等），Axis Properties 面板中**不显示** `Labels on Opposite Position` 相关配置
- 双轴 / 多 Y 轴：
  - 每个 Y 轴应**独立**管理自身的 label position
  - 当前实现对 “多于 2 根 Y 轴” 存在 bug（Issue #74033：设置后位置丢失/配置不生效）

## 2.3 数据 / 绑定规则（Data & Binding Rules）

- 轴上存在多个度量绑定（multi-binding）时：
  - `Labels on Opposite Position` 针对**该轴整体**生效，而非针对单个绑定
  - 删除/新增绑定不应重置 label position（当前实现：删除后位置仍保持）

## 2.4 多对象交互规则（Interaction Rules）

- 与 Chart Layout：
  - 切换标签位置会影响绘图区（plot area）尺寸与边距，需要重新布局
- 与 Export：
  - 导出（PDF/Image/Excel 图表）中的轴标签位置必须与设计器中保持一致
- 与 Bookmark / Storage：
  - 该属性作为轴配置的一部分，需参与序列化与反序列化（升级旧报表时应安全处理缺失字段）

# 4. Rendering & Layout Impact

> 仅在功能会影响可视化布局或渲染时保留本节；否则可省略。

**元素位置变化**

- 反转标签位置会导致：
  - X 轴标签从图表下方移动到上方（或反之）
  - Y 轴标签从左侧移动到右侧（或反之）
- 必须重新计算：
  - 轴与绘图区之间的边距（padding/margin）
  - 标签文本与 tick / grid line 的对齐

**多轴布局**

- 多 Y 轴（尤其是 3 根及以上 Y 轴）时：
  - 某一轴反转位置后，其他轴的对齐、交叠与可点击区域容易出错
  - 当前有已知问题：多 Y 轴场景下反转配置丢失、光标选中区域错位（Issue #74033）

**导出渲染**

- PDF / Image / Excel 图表导出使用渲染引擎的同一布局逻辑：
  - 经测试通路验证：导出的轴标签位置与 UI 一致
  - 仍需在大图、多图场景中关注文本是否被截断、重叠

# 5. Integration Impact

> 仅在功能对其他模块/能力有影响时保留本节；无集成影响可省略。

**Export / Print**

- 报表导出（PDF/图片/Excel 图表）需正确反映轴标签反转后的最终位置：
  - 已通过场景验证：导出结果目前与设计界面一致（📅 **测试-分析**: 通路验证，导出没有问题）
  - 仍建议在多图表报表、不同 DPI 下做 spot check

**Dashboard 运行态**

- Runtime 中，`Labels on Opposite Position` 是纯展示配置：
  - 不影响数据绑定与交互，仅改变标签绘制位置
  - 与 Selection / Filter 无直接关联

# 6. Script / Programmatic Support (TODO, 更新)

> 仅在功能支持脚本/编程控制时保留本节；纯 UI 功能可省略。

**脚本能力现状**

- 分析中已提及已有 Script Bug（Bug 73978），说明：
  - `AxisSpec` 的脚本接口（如 `setAxisStyle` / 未来的 `setLabelPosition`）存在历史问题
  - 当前特性主要通过 UI 控件封装，脚本支持尚未完全验证

**预期脚本契约（待开发/确认）**

- 读取：
  - `axis.getLabelPosition()` → 返回 `TOP/BOTTOM/LEFT/RIGHT` 或与 Axis Type 匹配的枚举
- 设置：
  - `axis.setLabelPosition(value)`：
    - 对于无效值（如 X 轴传入 LEFT），应抛出异常或拒绝并记录错误

**UI vs Script 优先级**

- 若脚本在运行时修改 label position：
  - UI 打开属性面板时应反映最新值
  - 再次从 UI 保存时，应覆盖脚本曾经写入但未持久化的值

> 📅 **测试-分析**：Script 相关目前以“存在历史 bug、待补充验证”为主，不作为本轮特性测试核心，但需在回归中留 1~2 条覆盖。

# 8. Regression Risk Areas

> 高度概括：仅保留最关键的回归关注点。

- **多轴布局**：多 Y 轴（尤其 3 根及以上）下的轴标签位置、命中区域、布局计算（含 Issue #74033）。
- **渲染一致性**：标签位置变化后，字体/旋转/网格对齐等不应被意外改变。
- **导出一致性**：导出结果（PDF/图片/Excel 图表）中的标签位置必须与设计器中一致。

---

# 9. Verification Strategy

> 高度概括：说明如何“最小代价”覆盖主要风险。

- **主线功能**：选 Bar / Combo / 多 Y 轴图，验证 X/Y 标签从默认侧切到对侧，并在保存 & 重新打开后仍保持。
- **组合场景**：在“多轴 + 多绑定 + 导出”组合下，快速检查布局是否错乱、导出是否一致。
- **兼容性抽样**：用 1～2 个旧报表做升级打开，确认默认标签位置不变即可。
