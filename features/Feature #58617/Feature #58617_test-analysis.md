## 一、需求分析（Requirement Analysis）

### 1. 功能理解与范围

**功能核心目标：**

-   在 Selection Container 元素的"添加过滤器"对话框中启用搜索功能
-   提供与表编辑中数据源列搜索功能类似的用户体验

**解决的业务问题：**

-   用户在添加过滤器时需从长列表中快速定位特定字段，搜索功能提升了可用性

**涉及模块：**

-   UI：Add Filter Dialog（add-filter-dialog.component）
-   UI 组件：Tree Widget（tree.component）
-   样式：Tree SCSS（tree.component.scss）

**功能类型：**

-   UI 交互增强 / 可用性特性

## 二、实现分析（Implementation Analysis）

### 1. 改动类型

**改动性质：** Feature（功能增强）

**影响层级：** UI 界面 + 样式

**具体改动：**

-   add-filter-dialog.component.html (Line 25)

``` html
[searchEnabled]="true"
```

为 `<tree>` 组件启用搜索功能

-   tree.component.scss (Line 29)

移除 `.search-box` 的固定高度 `height: 30px`

**目的：** 修复搜索框的渲染问题（可能是高度限制导致内容溢出或换行）

**受影响的用户路径：**

-   Add Filter Dialog 中的树组件搜索激活                           📅 **测试-分析**：影响Add Filter Dialog
-   所有其他使用 tree 组件带有 searchEnabled=true 的地方            📅 **测试-分析**：其它对话框没影响

------------------------------------------------------------------------

### 2. 需求实现一致性

  维度             评估   说明
  ---------------- ------ ----------------------------------------------
  核心功能完整性   完整   为 Add Filter Dialog 树启用搜索
  实现不足         存在   仅修改一行 HTML，依赖 TreeComponent.search()
  过度实现         否     无额外不需要的功能
  隐式行为变化     存在   移除 .search-box 高度限制影响所有树组件

示例代码：

``` html
<!-- wizard-binding-tree.component.html -->
<tree [searchEnabled]="true">

<!-- binding-tree.component.html -->
<tree [searchEnabled]="searchEnable">

<!-- asset-tree.component.html -->
<virtual-scroll-tree [searchEnabled]="searchEnabled">
```

------------------------------------------------------------------------

### 3. 关键实现风险

#### 风险 1：SCSS 改动缺乏完整影响分析

来源：

-   移除 `.search-box { height: 30px }`

影响：

-   所有树组件搜索框高度变化              📅 **测试-分析**：其它对话框没影响
-   浏览器之间高度差异                    📅 **测试-分析**：浏览器没影响
-   容器计算可能错误                      📅 **测试-分析**：忽略

可能后果：

-   Dialog 布局错乱                      📅 **测试-分析**：没影响
-   高度溢出或留白                        📅 **测试-分析**：没影响

------------------------------------------------------------------------

#### 风险 2：搜索状态未完全清除

``` ts
@Input()
set searchEnabled(searchEnabled: boolean) {
   this._searchEnabled = searchEnabled;
   if(!searchEnabled && !!this.searchStr) {
      this.searchStr = "";
      this.searchStart.emit(false);
   }
}
```

问题：

-   只清除 searchStr
-   不恢复树原始展开状态                📅 **测试-分析**：不影响，目前是全部展开   

------------------------------------------------------------------------

#### 风险 3：虚拟滚动搜索同步问题

``` ts
search(str: string) {
   ...
   setTimeout(() => {
      this.expandAll(this.root);
      this.dataSource.refreshByRoot(this.root, this.searchStr);
   }, 0);
}
```

影响：

-   UI 渲染竞态
-   大数据性能下降                  📅 **测试-分析**：可能影响性能

------------------------------------------------------------------------

#### 风险 4：树容器高度计算错误

``` ts
get treeContainerMaxHeight(): string {
   return this._searchEnabled ? "calc(" + maxH + " + 30px)" : maxH;
}
```

问题：

-   仍然使用固定 30px                  📅 **测试-分析**：目前看没影响

------------------------------------------------------------------------

#### 风险 5：多语言资源

``` html
placeholder="_#(Search)"
```

风险：

-   若缺少翻译资源，会直接显示 `_#(Search)`
