## script分析流程

### 第一阶段：让 Cursor 找到 script 相关逻辑

##### 推荐在 Cursor 里这样问

```markdown[script分析流程.md](script%B7%D6%CE%F6%C1%F7%B3%CC.md)
请帮我分析当前项目里 subject area 中的 script 标记相关的 prompt 以及代码逻辑：

1. 什么情况下 script 会被标记为 true 或者 false？
2. script=true 和 script=false 分别会影响哪些逻辑分支？
3. 请列出 subjectAreas prompt 中所有与 script 判断相关的规则
4. 请列出所有涉及 script 判断的代码文件路径 + 函数名

最后请输出为结构化 Markdown，保存到本地，方便我做测试分析。
```

👉 你必须让它「结构化输出」，否则后面不好喂给 ChatGPT。

### 第二阶段：让 Cursor 识别潜在风险点并提供测试方向

##### 在 Cursor 里问：

```markdown
我是一个 AI 测试工程师，我要对 prompt 中 script 识别的规则以及代码逻辑设计测试用例。

请你基于以上分析，帮我识别潜在测试风险点。

请按「风险描述 + 可能影响 + 建议测试方向」输出，文件保存到本地。
```

### 第三阶段：让 Cursor 生成 case

##### 将 script 知识喂给 Cursor，然后提问：

```markdown
生成一组高覆盖率测试用例，用于回归测试 script 的识别是否正确。

要求：
1. 覆盖不同的场景，验证 prompt 中所有 script 相关的规则
2. 每个 case 包含：
   - 场景
   - 用户问题
   - 预期 script 值
   - 设计意图 / 验证点
3. 每个测试用例必须仅聚焦于一个script触发规则
   避免在同一测试用例中组合多个与script相关的条件（例如，同时使用 runquery 和 function），因为这会导致无法隔离是哪个规则导致 script = true
4. 请生成至少 20 个英文 case。
```

### 第四阶段：Case Review

将 Cursor 的分析结果以及 prompt、背景知识，都扔给 ChatGPT，让 ChatGPT review 这些 case：

```markdown
你现在是一个资深 AI 测试架构师。

附件包含：
1. subject area 的 prompt
2. script 逻辑分析文档
3. 已生成的测试用例列表

请对这些 script 识别相关的测试用例进行严格 Review，并按以下步骤输出：

--------------------------------
【Step 1：覆盖度检查】

1. 是否覆盖 prompt 中 script 识别相关的各种规则
2. 输出覆盖矩阵：
   测试维度 | 是否覆盖 | 对应 case 编号 | 是否充分

--------------------------------
【Step 2：逐条 Case 质量评估】

对每一个 case：

- 合理性（1-5 分）
- 清晰度（1-5 分）
- 是否存在歧义

输出表格。

--------------------------------
【Step 3：Trigger Isolation Check】

请检查每个测试用例是否满足「触发条件隔离原则」。

定义：
如果一个测试目标是验证某个关键词 / 规则（例如 function），
则测试问题中不应同时包含其他足以独立触发相同结果的强信号（例如 script 关键字）。

请检查：
1. 是否存在多个 trigger 同时出现？
2. 是否存在「强触发词」掩盖了弱触发词？
3. 该 case 是否无法验证目标规则的独立效果？

对于存在问题的 case：
- 标记为【变量污染】
- 说明污染来源
- 提供一个修正版

--------------------------------
【Step 4：识别遗漏测试点】

列出缺失的测试维度，并说明为什么重要。

--------------------------------
【Step 5：优化建议】

对低质量 case：
- 重写优化版本
- 提供更强的测试表达方式
- 提供更清晰的 expected behavior

--------------------------------
【Step 6：新增高质量 Case】

基于当前分析，补充 10 个高价值回归 case：
- 包含风险类型标注
- 包含预期回答类型
- 包含验证重点

--------------------------------
【Step 7：输出】

将最终的所有 case 一起输出为表格的形式。
```
