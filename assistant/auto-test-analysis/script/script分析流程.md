## script分析流程

### 第一阶段：让 Cursor 找到 script 相关逻辑

##### 推荐在 Cursor 里这样问

```markdown
请帮我分析当前项目里 subject area 中的 script 标记相关的 prompt 以及代码逻辑：

1. 什么情况下 script 会被标记为 true 或者 false？请列出 subjectAreas prompt 中所有与 script 判断相关的规则
2. script=true 和 script=false 分别会影响哪些逻辑分支？请列出所有涉及 script 判断的代码的相关规则

最后请输出为结构化 Markdown，保存到本地，方便我做测试分析。
```

👉 你必须让它「结构化输出」，否则后面不好喂给 ChatGPT。

### 第三阶段：让 Cursor 生成 case

##### 将 script 知识喂给 Cursor，然后提问：

```markdown
根据前边的分析,生成一组高覆盖率英文测试用例，用于回归测试 script 的识别是否正确。

要求：
1. 覆盖 
   - prompt中 script 相关的规则
   - 多轮对话
     eg.第一轮用户问题，预期输出,当前轮用户问题，预期输出
   - 边界
2. 每个 case 包含：
   - caseID
   - 用户问题
   - 预期 script 值
   - 设计意图（只需要说明验证的具体规则内容，不要出现Rule 1, etc.）
3. 每个测试用例必须仅聚焦于一个script触发规则
   避免在同一测试用例中组合多个与script相关的条件（eg1.同时使用 runquery 和 function），因为这会导致无法隔离是哪个规则导致 script = true
4. 每条规则只能出现在一个测试用例中，每个测试用例只能验证一条规则，触发同一规则的不同措辞不应被视为不同的测试用例。
   eg."write a script" 和 "edit the existing script" 都是对script关键字明确提及的测试,只需要一条case
5. 所有的case显示在一个表格中,输出保存到md文件
```

### 第四阶段：Case Review

将 Cursor 的分析结果以及 prompt、背景知识，都扔给 ChatGPT，让 ChatGPT review 这些 case：

```markdown
你现在是一个资深 AI 测试架构师。

附件包含：
1. subject area 的 prompt
2. script 逻辑分析文档
3. 已生成的测试用例列表

请对这些 script 识别相关的测试用例进行严格 Review，并按以下步骤输出：

--------------------------------

【Step 0：测试意图识别】

对每个测试用例：

1. 判断想验证的 script 识别规则（Intended Rule）。
2. 列出用户问题中实际出现的 触发关键词（Trigger）。
3. 判断是否与设计意图一致，并在备注中说明问题。

输出表格示例：
CaseID	设计意图（Intended Rule）	实际触发 Trigger	是否一致	备注

--------------------------------
【Step 1：覆盖度检查】

1. 是否覆盖 prompt 中 script 识别相关的各种规则
2. 输出覆盖矩阵：
   测试维度 | 是否覆盖 | 对应 case 编号 | 是否充分

--------------------------------
【Step 2：逐条 Case 质量评估】
对每个测试用例，从 5 个维度评分（1-5 分）：

1. 测试意图清晰度
   问题是否能清楚表达要验证的规则
2. 触发强度
   是否能稳定触发目标规则
3. 触发隔离度
   是否避免无关 trigger 干扰
4. 语言自然度
   是否接近真实用户提问
5. 测试价值
   是否对回归测试有实际价值

输出表格示例：
CaseID	Intended Rule	意图清晰度	触发强度	隔离度	语言自然度	测试价值	问题/备注

--------------------------------
【Step 3：Trigger Isolation Check】

请检查每个测试用例是否满足「触发条件隔离原则」。

定义：
如果一个测试目标是验证某个关键词 / 规则（例如 function），
则测试问题中不应同时包含其他足以独立触发相同结果的强信号（例如 script 关键字）。

请检查：
1. 是否存在多个 trigger 同时出现？
2. 是否存在「强触发词」掩盖了弱触发词？
3. 该 case 是否无法验证目标规则的独立效果？

对于存在问题的 case：
- 标记为【变量污染】
- 说明污染来源
- 提供一个修正版

--------------------------------
【Step 4：识别遗漏测试点】

列出缺失的测试维度，并说明为什么重要。

--------------------------------
【Step 5：优化建议】

1.对低质量 case：
- 重写优化版本
- 提供更强的测试表达方式

2.补充遗漏的测试点

--------------------------------
【Step 6：输出】
1. 所有的分析过程输出在一个md 文件中
2. 最终的所有 case 单独输出一个md文件
```
