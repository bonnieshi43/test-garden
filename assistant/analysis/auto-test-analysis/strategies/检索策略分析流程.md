# 检索策略分析流程

## 第一步：让 Cursor 分析规则

在 Cursor 中选中这几个文件，然后提问：

> 请分析以下检索策略 prompt：
> 
> - default_strategies.prompt
> 
> 输出内容：
> 1. Rewrite 阶段的主要规则总结
> 2. Decomposition 阶段的主要规则总结
> 3. Context 是否注入上下文的判断逻辑
> 4. 这些规则可以转化成哪些测试维度
> 5. 哪些地方最容易出错
> 
> 用 markdown 输出，方便用于设计测试用例, 文件保存到本地

**目的：** 👉 让 Cursor 先理解策略规则

---

## 第二步：生成测试 case

最后问：

> 根据上面的规则和测试维度，生成一批用于测试检索策略的测试用例。
> 
> 要求：
> 1. 问题要像真实用户提问, 覆盖包含或者不包含stylebi功能的关键字的情况
> 2. ContextType 覆盖原则
>    - 设计测试用例时，应让不同的 contextType（dashboard、worksheet、freehand、table、chart、crosstab、portal、em、scheduleTask、dashboardPortal）在 case 中自然出现
> 3. 同时覆盖需要分解的复杂问题以及不需要分解的简单问题,但是不需要分解的简单问题不要超过case总数的六分之一
> 4. 测试用例应尽量覆盖常见疑问类型，包括但不限于 What、Why、How、Where
> 5. 覆盖多轮对话
> 6. 多规则覆盖原则
>    - 优先同时触发 rewrite 阶段和 decomposition 阶段的规则
>    - 设计时需明确标注该 case 所覆盖的每条规则，以便验证覆盖情况
>    - 若某些规则无法自然在同一问题中触发，则允许设计单规则的测试用例
> 7. 英文case占比80%

> 每个 case 包含：
> - Case ID
> - 用户问题
> - contextType
> - 预期 rewriteWithContext
> - 预期 allowContextInjection
> - 是否注入Context
> - 预期 final_queries
> - 验证的规则
>
> 所有case显示在一个表格中
> 生成md文件保存到本地
